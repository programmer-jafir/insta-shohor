Event loop is responsible for checking the call stack in the event. Queue also called as callback queue. If all the statements from the call stack have finished execution, then event loop will pop the statement from the event queue and push it to the call stack. Event loop is work asynchronously.

If you store data with local storage, that data will be permanent. That means even if you close the browser, it will still have data. And if you store any data with the help of session storage, the data will be will leave when the session is closed. That means if you close the browser, the data will be gone. This is the main differents between local storage and session storage.
